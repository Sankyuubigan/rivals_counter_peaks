# Debug / Тестирование без игры

Два способа смотреть и проверять интерфейс приложения без запущенной игры Marvel Rivals.

---

## Режим А. Быстрый просмотр UI в браузере (без Overwolf)

Самый простой способ «посмотреть интерфейс» прямо сейчас. Не требует Overwolf и Node.js.
Использует заглушку Overwolf API и локальный HTTP-сервер для загрузки базы данных.

### Запуск
1. Двойной клик по `overwolf_app/debug/serve.bat`.
2. Откроется браузер на странице `desktop-debug.html` (сервер на `http://localhost:8000`).
3. Сверху окна — панель отладки:
   - **ВРАГИ / СОЮЗНИКИ / БАНЫ** — кликай по иконкам героев, чтобы добавить/убрать (макс. 6 в поле).
   - **КАРТА** — выбери карту из списка (или «Без карты»).
   - **Язык** и флаги настроек трея (скрыть союзников, рейтинг, приоритет, избранные).
   - Кнопки: **Применить** (пересчитать UI), **Случайный матч** (авто-заполнение), **Очистить**.
4. После «Применить» вкладки «Контрпики» и «Тир-лист» пересчитываются через `marvelLogic`
   так же, как в реальной игре (`background.js: processGameData`).

### Остановка
Закрой окно консоли сервера (Ctrl+C) или просто закрой браузер и заверши процесс `python`.

### Файлы (не трогают рабочий код приложения)
- `debug/overwolf-mock.js` — заглушка `window.overwolf`.
- `debug/desktop-debug.html` — копия desktop-окна + подключение mock/логики/панели.
- `debug/debug-panel.js` + `debug-panel.css` — панель ручного ввода матча.
- `debug/serve.bat` — поднимает `python -m http.server` и открывает браузер.

> Примечание: в браузере не работают хоткеи Overwolf, drag окна и реальные игровые события —
> это только визуальная проверка UI и логики расчёта.

## Режим Б. «Честный» тест в Overwolf через Events Recorder & Player (ERP)

Официальный инструмент Overwolf для проигрывания записанной игровой сессии без игры.
Приложение запускается внутри реального Overwolf-клиента и реагирует на события как в игре.

### Подготовка (один раз)
1. Склонировать и собрать ERP (нужен Node.js):
   ```
   git clone https://github.com/overwolf/ow-events-recorder
   cd ow-events-recorder
   npm i
   npm run build
   ```
2. В Overwolf → Settings → Support → Development options → Load unpacked extension → выбрать папку `dist/` из ERP. Запустить ERP.
3. В этом проекте уже лежит `overwolf_app/manifest.debug.json` — копия манифеста, где
   все `"in_game_only": true` заменены на `false`, а `launch_events` переключён на
   `LaunchWithOverwolf`. Это нужно, чтобы in-game/notification окна показывались на десктопе
   при проигрывании записи. **Основной `manifest.json` не меняй — он для релиза.**
4. Загрузи `overwolf_app` как unpacked extension в Overwolf (через Packages → Load unpacked).
   При проигрывании ERP патчит приложение сам (добавляет `player.js` в background).

### Запись и проигрывание
1. В ERP нажми **Start recording**, запусти Marvel Rivals, сыграй матч, останови запись.
   (Запись можно сделать один раз и переиспользовать.)
2. В ERP → Play tab: введи UID твоего приложения (из окна Packages), выбери запись, нажми **Play**.
3. Приложение получает события (`onGameInfoUpdated`, `onInfoUpdates2`, и т.д.) и работает как в игре.

Подробности: https://dev.overwolf.com/ow-native/guides/test-your-app/events-recorder

## Что проверено
- Сервер отдаёт все ресурсы (HTTP 200).
- `logic.js` корректно загружает БД и считает контрпики/оптимальную команду.
- Все debug-скрипты проходят `node --check`.
- В `desktop-debug.html` присутствуют все DOM-id, нужные окну desktop.
